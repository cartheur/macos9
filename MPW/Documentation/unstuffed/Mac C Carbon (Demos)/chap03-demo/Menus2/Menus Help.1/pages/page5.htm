INDEXING HTML CONTENT

    Indexing your help content makes your content searchable by Apple Help. The Apple Help Viewer uses the Sherlock search engine to provide fast, full-text searching of Apple Help files.

    Indexing involves using the Apple Help Indexing Tool, which is included in the SDK, to create an index file.

Preparing to Index

    Before you create an index, you must ensure that your help files are in their final folder locations. (The index file stores relative path information, so moving content files after indexing invalidates the index file.)

Creating an Index

    To create an index file, drag and drop the folder containing your help content onto the Apple Help Indexing Tool. The Indexing Tool scans the folder and creates an Apple Help index file, which is saved inside the folder you dropped on the tool.

Controlling How Content is Indexed

    You can use the optional ROBOTS meta tag in the HTML headers in your content files to control how your content is indexed. For example:

    • Specifying a value of KEYWORDS tells the Indexing Tool to index only those terms specified with the KEYWORDS meta tag, leaving the rest of the content in that file unindexed.
    • Specifying a value of INDEX tells the Apple Help Indexing Tool to index the entire file as a possible search result.
    • Specifying a value of NOINDEX tells the Indexing Tool not to index the file. The Menus Help title page file, for example, specifies this value in a ROBOTS meta tag: 
         <meta name="ROBOTS" content="NOINDEX">
Adding Descriptions

    Using the optional DESCRIPTION meta tag, you can ensure that brief descriptive abstracts are displayed in Apple Help search results. For example, here is the description added to the Menus Help content file "page1.htm": 

       <meta name = "DESCRIPTION" CONTENT="This section 
   describes how to create a Help item in an
   application's Help menu.">
